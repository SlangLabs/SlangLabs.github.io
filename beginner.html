Nope, the title is not a clickbait, that's all it takes to get started and see Slang in action in an app. Let's get started and add Slang to an android app in less than 5 minutes. Follow the steps below:

What you should know

This codelab assumes you know how android works. If you are an experienced programmer and you are proficient in reading code, you will likely be able to follow this blog with ease.

What you will do

You will import an existing project in Android Studio, add Slang to it and perform actions inside the app using voice.

What you will need

To work through this codelab you will need a computer that can run Android Studio (or already has Android Studio installed) and of course an internet connection.

We will be using a demo app created by Slang called "Slang Travel" for this exercise.

Download the source code

Method 1: Using Git (if you are familiar with Git)

You can clone the repository using the following command in your terminal, to a directory of your choice.

git clone https://github.com/SlangLabs/slang_travel

Method 2: The longer method (if unfamiliar with Git)

  1. Go to the GitHub repository of the project by clicking here.
  2. On the right side click the Clone or Download button, and click Download ZIP option.

  1. Once the ZIP file downloads, unzip the contents into a directory of your choice.

Setting up the project in Android Studio

  1. Go ahead and open Android Studio.
  2. Click on the option Open an existing Android Studio project, and then navigate to the directory where you have downloaded the source code from GitHub.

  1. Once the app opens, wait for the Gradle build to finish.
  2. Once the Gradle build is finished, Go to app>java>SlangInterface File

Sign up & Sign in

Sign up to the Slang Console. This is required to get your API keys and Buddy Keys. Please follow the steps below to sign up for a Slang account.

Upon successful sign-up, you can sign into the Slang console using the appropriate credentials.

Now, let us get the keys required for the SDK integration. You can get the Buddy ID and API Key from the Slang Console.

API Key

The API Key is used to identify your account for any billing purposes. It's available via the "account" page of the Console as shown below.

Buddy ID from Travel.sample buddy

The Buddy ID uniquely identifies your app. On the console, open your buddy and look for the Buddy ID on the right side of the page. On signing up you will see travel.sample buddy created for you. You will find the buddy id there.

Publish ‘Travel.sample' buddy to Staging

Click on ‘Publish to Staging', this allows the SDK to know the environment to pull the schema from.

  1. Go to app>java>SlangInterface.

  1. Reach the file then search for BUDDY ID and API Key.
  2. On top of this file, you will observe the buddy_id and api_key marked as "TODO".
  3. Paste the API key and Buddy ID you copied previously here respectively.

Kudos! that's all, You have successfully integrated Slang in under 5 mins. Let's do the next step and should work now.

Now that you have integrated Slang, let's try it out. Currently, emulator sucks for testing Voice Interfaces so, we recommend using an actual mobile to try out Slang.

Go ahead and build the project on your mobile phone. You can search for trains in both English and Hindi.

Run Slang Travel on your mobile and now you can try saying:

"Show trains from Bangalore to Chennai"

"Trains leaving from Delhi to Mumbai on 12th July"

"Can you show me trains from Chennai on 1st August"
